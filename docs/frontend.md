## 0. 결정 사항 (2026-07-11 확정)

| # | 쟁점 | **결정** |
|---|---|---|
| **D1** | 그룹 정원 2명 vs N명 | ✅ **2인 유지.** 백엔드 정원 2 하드락 그대로 |
| **D2** | 그룹 특성(type) | ✅ **항상 커플 고정.** `couple/friends` 구분·그룹 타입 개념 **삭제.** 카운터는 항상 "사랑한 지 …" 톤 |
| **D3** | "함께 그림 그리기" 방식 | ✅ **턴제/이어그리기.** 기능정의서의 *소통-보내기 + 답장보내기* 그 자체. 실시간 공동 캔버스 아님 → **새 백엔드 불필요**, 기존 `POST /doodles`+`parent_id` 사용. 받은 그림을 캔버스 배경으로 깔고 이어그림 |
| **D4** | 매일 질문(Q&A) | ✅ **도입 안 함.** 자유 낙서편지만 |

**낙서(코어) 메커니즘 확정:**
> 낙서 = ① **펜으로 그림판처럼 직접 그리기**(펜 종류·색 선택) **또는** ② **이미지 업로드**(카메라/갤러리).
> 이걸 **편지처럼 상대에게 전송**하고, 상대는 **답장**으로 (받은 그림 위에 이어그리거나 새로) 되보낸다.
> "OO와 함께 그림 그리기"는 이 **주고받기(이어그리기) 루프**를 부르는 이름이지 별도 기능이 아니다.

### 0.1 백엔드 조율 없음 (확정)

**백엔드는 손대지 않는다.** 따라서:
- **그룹 타입 없음** — 항상 커플. `couple/friends` 분기 삭제.
- **관계 카운터(함께한 N일) 없음(pass)** — `groups.created_at`을 응답에 안 받으므로 D-day를 그리지 않는다.
  없는 날짜를 지어내지 않는다(나쁜 fallback 금지). 상단은 **상대 이름 ♥ 내 별명**까지만 표시하고 D-day 줄은 뺀다.

> 결과적으로 남은 백엔드 변경 요청은 **0개.** 앱은 현재 [API.md](API.md) v0.2 계약 안에서만 구현한다.

## 1. 컨셉 교정 (가장 중요)

**정정된 한 줄 정의:**
> **연인 둘이서 쓰는 "그림 편지" 앱.** 한 사람이 낙서를 그려 보내면 상대에게 **편지처럼** 도착한다.
> 편지를 주고받을 때마다 가운데 **펫이 먹을 간식이 생긴다.** — **그림 주고받기가 메인, 펫은 부차 시스템.**

- 지금 앱은 **펫 키우기가 홈의 주인공**처럼 되어 있는데, **순서를 뒤집어야 한다.** 홈의 중심은
  "받은 편지 / 그림 주고받기"이고 펫은 그 결과로 자라는 반려 존재다.
- **받은 그림을 보는 경험이 특히 중요.** 지금은 "토리가 나에게 그림을 그리게 하는" 쪽에 무게가
  실려 있는데, **상대가 보낸 그림이 어떤 모양인지 보는 것**을 최소한 그만큼 강조해야 한다.
- 타깃은 **커플 2인 고정**(친구용 그룹 타입은 도입 안 함 — D2).

## 3. 새 홈 화면 IA (썸원 참고 · 무스크롤 단일 화면)

```
┌───────────────────────────────┐
│  [간식n]      Sumone(로고)   [알림][ㅁ] │  ← 상단바: 재화 · 워드마크 · 알림
│         상대이름 ♥ 내별명            │  ← D-day 줄 없음(카운터 pass)
│                                     │
│            [받은 편지 ●1]            │  ← 새 낙서/편지 알림(있으면) → 뷰어
│                                     │
│     ┌───── 집 배경 ─────┐          │
│     │      🚫이모지금지            │
│     │     [ 펫 캐릭터(손그림) ]     │  ← 탭=쓰다듬기, 집 배경 위에
│     └───────────────────┘          │
│      "OO이와 함께 그림 그리기"        │  ← 코어 진입(그리기/함께그리기) — D3
│                                     │
├───────────────────────────────┤
│  [편지]   [그림함]   [일기]   [커플]  │  ← 아이콘 하단 네비(무텍스트/최소텍스트)
└───────────────────────────────┘
```
- **바로가기 그리드 제거**(C2). 스토어·레포트·집꾸미기·설정은 각 탭 안이나 상단 메뉴로.
- 하단 네비 4탭(안): **편지(소통/그리기)** · **그림함(사진첩)** · **일기(펫일기/레포트)** · **커플(펫·집·설정)**. → 탭 구성은 확정 필요(§0 이후).

## 7. 액션 아이템 (구현 상태)

1. ✅ **썸원 디자인 토큰 확정** — 레퍼런스 4장 분석 → 크림 배경·웜 브라운잉크·하트핑크·골드·세리프 워드마크·둥근 radius(theme.dart)(§4).
2. ✅ **셸 리스킨** — 배경/타이포/둥근 컴포넌트/아이콘 하단네비(CpNavItem), **이모지 전면 제거(app/lib 0건)**(B).
3. ✅ **홈 IA 재구성** — 4탭 아이콘 네비[홈·그림함·일기·더보기], 무스크롤 단일화면, 펫 중심+집 배경+받은편지 카드+함께그리기 CTA, 바로가기 그리드 제거(§3).
4. ✅ **낙서·이어그리기** — 펜 그림판 + 이미지 업로드 → 편지 전송 → 답장 시 받은 그림을 배경으로 이어그림(D3, 기존 API).
5. ✅ **펫 아이템 시스템** — 손그림 벡터 아이템 8종(charlab/items), 상점 썸네일(ItemThumb) + 펫 위 착용(PetItemLayer) + 종별 앵커(charlab/anchors)(E).
6. ✅ **집 시스템** — 홈에 펫 뒤 방(집) 배경(_RoomPainter) + 집 꾸미기 화면(가구/소품 pos 배치)(E4).
7. ✅ **내부 구조 정합** — 모델 v0.2(BestDoodle·WidgetData·dominantType) + RestRepository/SocketRealtime + --dart-define=API_BASE 스위치(F1/F2).
8. ⏳ **피그마 재감사** — 필요 시 화면 세부를 피그마와 재대조(C4/C5). (남은 항목, 데모엔 비필수)
9. ⏳ **아이템 아트 정교화** — 현재 벡터 아이템은 단순 버전. subagent QA 루프로 더 예쁘게 다듬기 가능(선택).

> 관계 카운터·그룹 타입은 **제외**(백엔드 무변경). 남은 백엔드 요청 0개.
> 실서버 연결: `flutter run --dart-define=API_BASE=https://anjonghwa.madcamp-kaist.org` (없으면 오프라인 Mock 데모).

## 7.5 커플감(로맨스) 리팩토링 — "가족/키즈 느낌" 피드백 반영 (v0.4)

리스킨 후에도 "커플보다 가족 느낌"이라는 피드백을 받아 원인을 진단하고 고쳤다.

| 원인 | 조치 |
|---|---|
| 펫이 홈의 주인공(화면 40%가 펫+아기방) | 홈 히어로를 **받은 편지**로 교체 — 상대의 실제 낙서/사진/손글씨 텍스트를 편지 카드로 크게. 펫은 104px 동반자로 축소, 방(_RoomPainter) 제거 → 그림자 한 줄 |
| 캐릭터가 유아용(두꺼운 선·광택 눈·볼터치·파티모자 기본) | toolkit 전역: 선굵기 ×0.62·눈 광택 기본 off·볼터치 0.30, 시드에서 착용 해제 |
| 로맨스 장치 부재 | 세리프 워드마크(Memory Pager) + 세리프 커플 라인 + **손글씨 폰트 번들**(나눔펜, OFL) — 편지·펫 대사·캡션에 사용 + **턴 상태 줄**("내가 답장할 차례" / "○○의 답장을 기다리는 중" — 앨범에서 정직하게 유도) |
| 카피가 아이 말투 | 펫 대사·CTA 전면 교체(커플 문맥: "둘이 보내준 간식…", "○○에게 마음 그리기") |
| tofu(□) 버그 | 이모지 제거 잔여물(변형선택자·ZWJ) 전면 삭제 |
| 원색 아이템 | 아이템 팔레트 뮤트(더스티 로즈·샌드 등) |

D-day류 카운터는 계속 제외(goal 제약). 미열람 사라지기 편지는 홈에서도 봉인 표시(내용 비노출).
