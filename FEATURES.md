# Memory Pager — 구현 상태 (정직판)

> 커플(2인) 전용 폐쇄형 소셜 앱. 사진·손낙서를 삐삐 감성으로 주고받고,
> 함께 키우는 펫이 낙서를 보고 그림 일기를 그려준다.
> Flutter(Android) + FastAPI + Socket.IO + MySQL + GPU(EXAONE LLM / Stable Diffusion).
>
> **이 문서는 2026-07-13 외부 감사 후 전면 교정한 것이다. 이전 판은 "엔드포인트 배선 + happy-path
> E2E"만 보고 ✅를 과장했다.** 상태는 실제 동작 기준으로 다시 매겼다.
>
> 상태: ✅ 실서버 동작 검증 · 🟡 부분/제약 있음 · 🔴 미동작(정적 UI/미구현).

---

## ⚠️ 먼저 — 반드시 알아야 할 것
1. **release APK는 방금 전까지 서버 통신 불가였다.** `main/AndroidManifest.xml`에 INTERNET 권한이
   없어 debug/profile에만 있었다. → **수정 완료.** 단 **기존 CI 아티팩트(그 이전 커밋)는 폐기**하고,
   INTERNET 권한이 포함된 **새 APK를 다시 빌드**해야 한다.
2. **백엔드와 프론트가 다른 브랜치였다.** 질문 API·join 응답 래핑은 `anjonghwa` 브랜치에만 있었다
   (배포 서버는 anjonghwa라 라이브는 됐음). → **anjonghwa를 front-remake로 머지 완료.**
   이제 front-remake 하나로 전체 시스템 재현 가능. (최종적으로 main 통합은 별도 결정 필요)
3. **실기기 검증은 아직 안 됐다.** 헤드리스 구동이 막혀 웹 미리보기 + API E2E로만 검증했다.
   갤럭시에 새 APK를 깔고 2인 실사용 확인이 남았다.

## 백엔드 (배포 서버, anjonghwa=front-remake) ✅ — 거의 완성
- 인증·기기 등록, 그룹 생성/참여/상세/수정, 멤버 별명.
- 낙서(그림/텍스트/사진) 업로드·조회·확인(view)·사라지기(ephemeral)·답장(parent_id).
- 펫 조회/쓰다듬기, 찌르기, 오늘의 질문(E-1), 그림일기 목록, 월간 레포트(+생성), 홈 위젯.
- Socket.IO `/rt` 실시간 이벤트.
- GPU 연동: 낙서→일기 이미지(SD), 펫 활동/대사(LLM), 레포트.
- 통합 테스트 다수 그린.

## 🔴 항목 처리 결과 (2026-07-13 목표: 빨간 항목 전부 해결)

### 해결 완료 ✅ (analyze 0 · 골든 포함 테스트 23/23 통과)
- **스토어 코인 게이팅(U-1)** — 미보유 아이템 구매(코인 부족 시 안내), 카테고리 선택, 저장. *구매는 세션 로컬(서버 구매 API 없음).*
- **펫 카테고리/저장 버튼** — 카테고리 전환 + 저장 확정 동작.
- **뷰어 하트** — 로컬 좋아요 토글(회색↔코랄). *서버 리액션 API 없음.*
- **이웃 검색** — 코드 입력 필드 + 엔터/버튼 제출 + 피드백. *백엔드 이웃 방문 API 없음.*
- **달력 월 이동·날짜 선택** — 실제 요일 그리드로 StatefulWidget 재구현.
- **레포트 월 이동** — `reportMonths`/`report(month)` 로 있는 달 사이 이동.
- **텍스트 낙서 전송 UI** — 캔버스 T 버튼 → 입력 → `sendText`(엔터·실패 스낵바·parent_id).
- **온보딩 별명 단계** — 실서버 참여자 흐름에 연결(중복 생성 방지 가드).
- **앨범 배너** — 그림일기 진입 연결.
- **사진 카메라/갤러리** — image_picker(카메라/갤러리/제거). 전송 PNG에 배경 사진 합성.
- **로그아웃·커플 연결 끊기** — `POST /groups/{id}/leave` + `mock.logout`. **✅ 라이브 배포+E2E 완료**(leave 204 → /me group:null → 온보딩).
- **낙서 AI 캡션** — ✅ **실제 구현·배포·E2E 완료.** BLIP(이미지→영어 서술) + EXAONE(펫 말투 한국어) 파이프라인. GPU `/caption` 엔드포인트 + 백엔드 백그라운드 생성 + `caption` 컬럼(라이브 마이그레이션). 실측: 사진 낙서 업로드 → 4초 → *"하얀 드레스 입은 여인이 풀숲에 앉아있다니, 마치 동화 속 같네요!"*
- **LoRA 화풍 학습** — ✅ **실제 구현·배포·풀스택 E2E 완료.** 커플 손그림으로 SD 1.5 LoRA(UNet attention, rank8)를 학습해 펫 일기를 그 화풍으로 그린다. GPU `/train/style`(학습→6.4MB 가중치 저장, 300step ~64s, VRAM 여유) + learned `/generate/diary`(어댑터 로드·트리거 프롬프트). 백엔드 `train_learned_style`(손그림 수집→`style_models` training→ready) + 손그림 20장 자동 트리거 + 데모용 수동 `POST /groups/{id}/style/train`(5장). `_active_style`이 ready learned 를 우선하므로 이후 새 일기부터 자동 적용. **라이브 검증:** 그림 6장 업로드 → 학습 트리거 → 76초 후 `ready`(weights_path 기록) → 백엔드 프로덕션 경로(`_active_style`→GPU)로 learned 일기 렌더 성공. *단, 검증은 합성/단순 손그림 기준 — 실커플의 다양한 낙서 20장으로 화풍·펫 내용 균형(step·rank·캡션 튜닝)은 실데이터가 쌓여야 최종 확인된다.*

### 남은 것 🔴
- **FCM 푸시** — `firebase_messaging`은 **Firebase 프로젝트(google-services.json)** 없이는 빌드가 깨진다(외부 계정·앱 등록 필요). 백엔드 `/v1/devices`(fcm_token)는 준비됨. → Firebase 프로젝트 생성이 선행돼야 함. *유일하게 외부 계정이 필요한 항목 — 이제 남은 유일한 🔴.*
