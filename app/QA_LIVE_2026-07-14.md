## 발견 사항 (누적)

1. ⚠️ **온보딩 이름입력**: 키보드가 올라오면 "시작하기" 버튼이 헬퍼 텍스트("상대방에게 보여질 이름이에요")를 덮는다. 추정: 키보드 회피로 버튼을 밀어올릴 때 헬퍼 텍스트와 z-겹침. 경미.
2. ⚠️ **온보딩 그룹생성 화면**: "그룹 만들기" 카드 본문 안내와 바로 아래 점선 박스가 "그룹을 만들면 초대 코드가 생성/생겨요" 를 **중복** 안내. 하나로 합치는 게 깔끔.
3. ✅ **#2 그룹생성 → #23 대기화면**: 초대코드 표시(7FJM33FA)·복사 버튼·"복사했어요" 토스트·스피너·취소 정상.
4. ✅ **#23 자동전환**: 서버로 B 참여시키자 A 기기 대기화면이 홈으로 자동 전환. 폴링 동작 확인.
5. ✅ **홈**: 그룹명/D+1/펫(모리 Lv.1)/낙서보내기/콕찌르기(오늘 3번)/오늘의 질문 정상 렌더.
6. ✅ **오늘의 질문 답변**: 홈 입력 탭 → 모달 → 남기기 → 카드에 저장·표시. 정상.
7. ✅ **#3 그리기 캔버스**: 획 그리기·도구패널(연필/붓/마커)·HSV 색상환 확장 피커·명도 슬라이더·색 변경 즉시 반영. NaN 크래시 없음.
8. ✅ **낙서 전송 → #9 실시간 + FCM**: A 전송 후, 서버로 B 낙서 보내자 A 홈에 새 낙서 카드 실시간 추가 + FCM 헤즈업("새 낙서 도착 · 나무(B)님이 낙서를 보냈어요"). 전송 버튼에 파트너 실명("나무(B)에게 보내기") — #17 유령파트너 없음.
9. ✅ **받은 낙서 상세 + 하트**: 손글씨 폰트 전문 표시, 하트 좋아요 스낵바("이 낙서를 좋아해요 ♥") + 버튼 채워짐.
10. 🐛/⚠️ **받은 낙서 뷰어의 "!"(찌르기) 버튼 무피드백 + 모호한 아이콘**: viewer.dart:329 가 `mock.poke` 를 호출하지만 하트와 달리 스낵바/확인이 전혀 없어 **눌러도 아무 반응 없는 죽은 버튼처럼 보임**. 아이콘(`_PokePainter`, 느낌표 모양)은 "신고/경고"로 오인됨 — 이게 콕 찌르기인지 알 수 없음. 추가로 mock.poke(mock.dart:539)는 API 에러를 `catch(_){}`로 삼키고 `pokesToday`를 낙관적 선증가 → 서버가 하루 3번 제한 초과로 거절해도 유저는 모르고 클라 카운트만 어긋남. **원인**: 피드백/아이콘 시맨틱 누락 + 낙관적 카운트. 재검토 대상.
11. ⚠️ **홈 펫 말풍선**: 낙서를 새로 받은 직후에도 말풍선이 "오늘도 낙서 기다리는 중… 삐삐!" 그대로 — 방금 도착했는데 "기다리는 중"이라 약간 모순. 경미.
12. 🐛 **답장 화면 "받은 낙서 보기" 미리보기가 항상 데모 스톡 사진(대나무밭 여성)** — 실제 답장 대상 낙서와 무관. **원인**: draw_canvas.dart:457-458 이 네트워크 대응 렌더러 `doodleImage()`(mock.dart:882, network↔asset 자동선택)를 안 쓰고 `Image.asset(widget.replyTo!.asset ?? 'assets/photos/photo_field.png')` 를 직접 사용. `asset` 은 데모 전용 필드(mock.dart:33)라 실서버 낙서에선 항상 null → 번들 데모 사진으로 폴백. 텍스트 낙서면 본문도 안 보임. 실사용 시 "상대가 보낸 낙서에 답장"인데 모르는 사람 사진이 떠서 혼란. 탭해도 무반응. **수정 방향**: 미리보기를 `doodleImage(widget.replyTo!)` 로 교체(텍스트 낙서는 본문 썸네일 렌더).
