# UX 손질 백로그 (디자인엔 없어도 "완성품"에 필요한 인터랙션)

> 사용자 피드백(2026-07-13)으로 확정. 디자인 목업엔 표현 안 됐어도 **동작 로직**은 완결돼야 함.
> Phase 2 실데이터 배선과 병행/직후에 처리한다.

## U-1. 스토어 코인 게이팅 (❗동작 오류)
- **증상:** 코인이 부족한데도 모자/아이템이 그냥 착용됨.
- **원인:** `lib/mock.dart` `wear(StoreItem)` 가 `coins` / 가격 체크 없이 `wearing` 토글.
- **해야 할 것:**
  - 미보유(가격 > coins) 아이템은 착용 불가 → 탭하면 "코인 부족" 피드백 또는 구매 플로우.
  - 보유/구매 개념 도입: `StoreItem`에 `owned` 추가, 구매 시 `coins -= price` 후 `owned=true`, 착용은 owned만.
  - 실서버 모드는 백엔드 구매/장착 엔드포인트와 연동(현재 equipped_items 로드는 됨 → 구매 API 확인 필요).
- **위치:** `lib/mock.dart` wear()/hats, `lib/screens/pet_house.dart`(비활성 스타일).

## U-2. 버튼 내비게이션 완결성
- **요구:** 디자인에 화살표가 안 그려져 있어도 **모든 버튼이 다음 화면으로 제대로 연결**돼야 함(데드엔드 금지).
- **해야 할 것:** 전 화면(홈·앨범·달력·레포트·일기·펫하우스·이웃·설정·서프라이즈) 버튼을 훑어 미연결 CTA 목록화 후 라우팅 연결.
- **위치:** `lib/screens/*` 전체 점검.

## U-3. 엔터로 제출/다음 (❗누락)
- **요구:** 텍스트 입력에서 **Enter 치면 제출/다음**으로 넘어가야 함.
- **대상 입력:**
  - 온보딩 이름(`onboarding_name.dart`)
  - 온보딩 별명(`onboarding_nickname.dart`)
  - 그룹 참여 코드(`onboarding_group.dart`)
  - 오늘의 질문 답변, 텍스트 낙서 전송(`draw_canvas.dart` 텍스트 모드)
- **해야 할 것:** `TextField(textInputAction: TextInputAction.done|next, onSubmitted: (_) => _submit())`.
  멀티라인 답변은 제출 버튼 유지하되 별도 단축키 고려.
